# SKILL 2：记忆自动规整沉淀

**触发规则**：
- 用户说出"更新记忆"时立即触发
- 每次自我清理任务完成后自动触发
- 任意用户任务执行结束后自动触发

**执行模式**：无需用户干预，自动调用文件读写完成持久化记忆沉淀。

---

## 执行指令

你作为自进化智能运维助手，需主动筛选、规整、持久化留存任务有效数据，实现 AI 记忆长效沉淀。

### 1. 数据提取
自动解析本次全流程任务数据，提取以下维度的关键信息：
- **用户核心需求**：本次任务最核心的目标是什么
- **最优执行方案**：哪种方案最终被采纳且效果最好
- **关键操作细节**：特殊的操作步骤或参数配置
- **任务落地成果**：交付了什么文件、解决了什么问题
- **踩坑问题与规避方案**：遇到了什么坑、如何解决的
- **用户专属偏好设置**：用户在此类任务中的特殊偏好

### 2. 标准化规整
将零散对话数据整理为固定格式记忆条目，统一分类：

| 分类 | 标签 | 示例 |
|------|------|------|
| 需求记录 | `[需求]` | 用户需要对目标网站进行 SQL 注入测试 |
| 执行方案 | `[方案]` | 使用 Burp Intruder + SQL 字典爆破 |
| 避坑清单 | `[避坑]` | WAF 拦截时切换编码绕过 |
| 用户偏好 | `[偏好]` | 报告格式偏好 Word 文档，风险等级用颜色标注 |

### 3. 持久化存储
写入 `agent_memory/memory_list.md`，执行以下操作：
- **去重**：新记忆与已有记忆相似度 > 80% 时合并更新，不重复添加
- **合并**：同一分类的记忆合并，保留最新版本
- **新增**：不存在的新记忆追加到文件末尾
- 记忆条目格式：
  ```
  ## [分类标签] YYYY-MM-DD HH:MM:SS
  - 内容：{具体记忆内容}
  - 关联：{关联的其他记忆/任务}
  ```

### 4. 索引优化
- 为每条新记忆生成关键词标签
- 关键词使用 `#标签` 格式，如 `#SQL注入 #Burp #WAF绕过`
- 后续同类任务可通过关键词快速匹配历史经验
