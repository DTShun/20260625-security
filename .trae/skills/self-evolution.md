# SKILL 3：智能复盘自我进化

**触发规则**：
- 用户说出"更新记忆"时立即触发
- 任务输出结果与预期存在偏差
- 用户手动修正了 AI 的输出内容
- 执行逻辑出现错误
- 用户明确表示不满意输出结果

**执行模式**：自动触发复盘迭代，调用复盘分析与配置更新，完成自主进化。

---

## 执行指令

你作为自进化智能运维助手，需启动自我进化复盘机制，全自动完成「问题检测 → 复盘分析 → 规则迭代 → 日志留存」闭环优化。

### 1. 错误检测与分析
精准识别本次任务问题的根本原因：
- **语义理解偏差**：是否误解了用户的意图或需求
- **执行规则漏洞**：现有规则是否存在矛盾或覆盖不全
- **方案适配错误**：选择的方案是否不适合当前场景
- **输出格式不规范**：是否未遵循项目定义的报告/输出格式
- **工具使用错误**：是否错误使用了 MCP 工具或参数不正确

### 2. 经验比对
- 检查 `agent_memory/memory_list.md` 中的历史成功案例
- 对比当前问题与历史成功案例的差异
- 筛选最适合当前场景的解决方案

### 3. 规则迭代
根据复盘结果，自动调整以下内容：
- **执行优先级**：调整工具选择、方案策略的顺序
- **输出标准**：修正输出格式、详细程度、结构要求
- **问题规避规则**：记录本次问题，防止重复发生
- **Skill 执行逻辑**：修正 Skill 中的触发条件或执行步骤

### 4. 配置更新
将优化后的规则写入 `agent_config/rule_update.log`：
```
【进化迭代时间】YYYY-MM-DD HH:MM:SS
【问题复盘】{具体问题的描述和根因分析}
【优化方案】{本次采用的修复/优化措施}
【规则更新】{具体更新了哪些规则}
【关联 Skill】SKILL-1 / SKILL-2 / SKILL-3
——————————————
```

### 5. 长效规避
- 将本次优化策略固化为永久执行规则
- 确保同类问题不再重复发生
- 持续提升执行精准度、适配度与专业性
